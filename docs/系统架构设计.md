# 系统架构设计

## 1. 背景

招聘流程中，人工筛选大量 PDF 简历耗时较高，并且不同招聘人员对候选人的判断标准可能不一致。本系统通过 PDF 解析、AI 信息抽取和岗位匹配评分，将非结构化简历转化为结构化候选人信息，并输出可解释的匹配结果。

## 2. 架构目标

- 支持单个 PDF 简历上传与多页解析
- 通过 AI 模型提取简历关键信息
- 支持输入岗位 JD 并计算匹配度
- 使用 JSON 返回结构化结果，方便前后端联调
- 使用缓存减少重复解析和重复评分
- 适配阿里云 Serverless 部署，降低运维成本

## 3. 总体架构

```text
浏览器前端
   |
   | 上传 PDF / 输入 JD
   v
API 网关或 FC HTTP 触发器
   |
   v
FastAPI 后端服务
   |
   |-- PDF 解析服务：提取多页文本
   |-- 文本清洗服务：去除乱码、冗余空白、页眉页脚
   |-- AI 抽取服务：提取姓名、电话、邮箱、学历、项目经历等
   |-- JD 分析服务：提取岗位关键词和要求
   |-- 匹配评分服务：计算规则分和 AI 语义分
   |-- 缓存服务：缓存简历解析结果和匹配结果
   |
   |-- Redis / Tair
   |-- DashScope 大模型 API
   |-- OSS 文件存储，可选
```

## 4. 后端模块设计

### 4.1 API 层

负责处理 RESTful 请求、参数校验、异常转换和统一响应格式。

主要接口：

- `POST /api/v1/resumes/upload`
- `POST /api/v1/resumes/{resume_id}/extract`
- `POST /api/v1/match`
- `GET /api/v1/results/{task_id}`
- `GET /api/v1/health`

### 4.2 PDF 解析模块

优先使用 `PyMuPDF` 提取 PDF 文本。若解析文本为空或质量较差，再使用 `pdfplumber` 兜底。

处理逻辑：

- 校验文件类型为 PDF
- 限制文件大小，避免异常大文件影响函数执行
- 遍历所有页面提取文本
- 合并页面文本并保留合理段落
- 生成 `resume_hash`，用于缓存和去重

### 4.3 文本清洗模块

处理目标：

- 合并重复空白字符
- 去除不可见字符和明显乱码
- 规范换行与段落
- 保留邮箱、电话、项目经历等关键信息

### 4.4 AI 信息抽取模块

模型输入为清洗后的简历文本，输出固定 JSON 结构。

抽取字段：

- 姓名
- 电话
- 邮箱
- 地址
- 求职意向
- 期望薪资
- 工作年限
- 学历背景
- 项目经历
- 技能关键词

电话和邮箱优先使用正则提取，AI 结果作为补充和纠错。

### 4.5 匹配评分模块

匹配评分由规则评分和 AI 评分组成。

规则评分：

- 技能关键词匹配率
- 工作年限匹配度
- 学历要求匹配度
- 项目经验相关性

AI 评分：

- 岗位职责与候选人经历相关性
- 项目经历与业务场景相关性
- 综合岗位契合度

最终评分：

```text
total_score = rule_score * 0.7 + ai_score * 0.3
```

## 5. 缓存设计

缓存分两类：

- 简历解析缓存：`resume:{resume_hash}`
- 匹配评分缓存：`match:{resume_hash}:{jd_hash}`

缓存内容：

- PDF 提取文本
- 清洗后的简历文本
- AI 抽取结果
- JD 分析结果
- 匹配评分结果

默认过期时间建议为 24 小时到 7 天。

## 6. Serverless 部署设计

后端部署到阿里云函数计算 FC：

- 使用 HTTP 触发器暴露 RESTful API
- 函数内运行 FastAPI 应用
- 环境变量存储模型密钥、Redis 地址、OSS 配置
- 对 PDF 文件大小、请求超时时间和并发进行限制

前端部署到 GitHub Pages：

- 使用 Vite 构建静态资源
- 通过环境变量配置后端 API 地址
- 页面直接访问线上后端接口

## 7. 异常处理

常见异常：

- 文件不是 PDF
- PDF 无法解析文本
- AI 模型调用失败
- JSON 解析失败
- Redis 连接失败
- 请求参数缺失

处理方式：

- 返回统一错误码和错误信息
- AI 调用失败时保留基础解析结果
- Redis 失败时不影响主流程，只跳过缓存

## 8. 性能优化

- 使用文件 hash 避免重复解析
- 对 AI 抽取结果做缓存
- 限制 PDF 文件大小
- 对长文本做截断或摘要后再传入模型
- 使用结构化 prompt 降低 JSON 解析失败率
