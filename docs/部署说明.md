# 部署说明

## 1. 部署目标

本项目采用前后端分离架构。当前版本已完成本地演示闭环，后续可以平滑迁移到云端：

- 本地演示：FastAPI + Vite
- 后端云端目标：阿里云函数计算 FC
- 前端云端目标：GitHub Pages 或其他静态托管
- 缓存：MVP 阶段使用内存缓存，生产环境建议替换为阿里云 Tair
- 文件存储：MVP 阶段使用内存存储，生产环境建议替换为 OSS

## 2. 后端部署到阿里云函数计算

### 2.1 准备工作

需要提前准备：

- 阿里云账号
- 函数计算 FC 服务
- 阿里百炼 DashScope API Key
- 可选：Redis / Tair 实例
- 可选：OSS Bucket

### 2.2 环境变量

在函数计算控制台配置：

```bash
DASHSCOPE_API_KEY=你的阿里百炼 API Key
MODEL_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL_NAME=qwen-plus
MODEL_NAME_FAST=qwen3.5-flash
MODEL_NAME_PRECISE=qwen3.5-plus
MODEL_NAME_PRECISE_OCR=qwen-vl-ocr-latest
PRECISE_MAX_OCR_PAGES=3
MAX_FILE_SIZE_MB=10
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

本地调试时也可以在前端顶部输入“百炼 Key”，点击“生成 env”，后端会自动生成 `backend/.env`。该文件已加入 `.gitignore`，不会提交到 GitHub。

### 2.3 函数配置建议

| 配置项 | 建议值 |
| --- | --- |
| 运行时 | Python 3.10 或 Python 3.11 |
| 内存 | 512 MB 到 1024 MB |
| 超时时间 | 60 秒 |
| 触发器 | HTTP 触发器 |
| 请求方法 | GET、POST、OPTIONS |
| 单文件大小限制 | 10 MB |

### 2.4 部署步骤

1. 本地安装依赖并确认后端可运行
2. 将后端代码和依赖打包
3. 在阿里云函数计算创建 Python 函数
4. 上传代码包或使用 Serverless Devs 部署
5. 配置环境变量
6. 创建 HTTP 触发器
7. 测试 `/api/v1/health` 接口

### 2.5 本地调试步骤

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

```bash
cd frontend
npm install
npm run dev
```

打开：

```text
http://127.0.0.1:5173
```

## 3. 前端部署到 GitHub Pages

### 3.1 配置 API 地址

在前端项目中配置后端 API 地址：

```bash
VITE_API_BASE_URL=https://你的函数计算域名/api/v1
```

### 3.2 构建前端

```bash
cd frontend
npm install
npm run build
```

### 3.3 部署方式

推荐使用 GitHub Actions 自动部署到 GitHub Pages。

部署完成后，需要在 GitHub 仓库设置中开启 Pages，并选择对应分支或 Actions 作为发布来源。

## 4. CORS 配置

后端需要允许前端域名跨域访问。

开发阶段可以允许：

```text
http://localhost:5173
```

线上阶段建议只允许：

```text
https://你的用户名.github.io
```

## 5. 演示前检查清单

- 前端页面可以正常打开
- `/api/v1/health` 返回正常
- 可以上传 PDF 文件
- 可以解析多页 PDF
- 可以输入 JD 并返回匹配评分
- 可以看到 AI 简短总结
- 顶部百炼 Key 调试入口可生成 `.env`
- README 中的 GitHub 仓库地址已补充
- README 中的线上演示地址已补充
- 准备好 `docs/test-pdfs` 中的测试简历和 1 份测试 JD

## 6. 成本说明

本项目不需要购买长期服务器。阿里云函数计算按调用量和执行时长计费，面试演示流量很小，成本通常很低。

如需缓存，可以使用阿里云 Tair 小规格实例；如果时间紧张，也可以先实现内存缓存，并在 README 中说明生产环境建议替换为 Redis / Tair。
